vidview

vidview (from VID = 6D = connexel data) is a viewer for the exploration of connectivity data. This data is comprised of connections strengths between two positions in space, and can be derived e.g. from functional connectivity resting state fMRI measurements.

Compiling:

The viewer is programmed using the Qt toolkit. The easiest way to compile the viewer is using QT creator (see http://qt.nokia.com/products/developer-tools/). It is also possible to compile using qmake / make.

The only other external dependency is the boost program_options library (libboost_program_options). In addition to the library, the include files have to be accessible. The project (.pro) file expects these files at /usr/lib and /usr/include, but this can be edited in case the files are not available there on the target system.

Successful compilation results in an executable called vidview. For this documentation, it is assumed that this files is included in the search path of the computer that the software is run on.

General use example:

vidview --surfsetl data/surfaces/lh.set --scons data/connections/left_cons_thr0.6_roi_vlpfc_cons.txt

This example assumes that vidview is executed in the source directory, containing the example data in “data”.

Surface files (--surfsetl / --surfsetr):

This parameter points to one or two files with a list of surface and normal files created with Freesurfer. Right now, surfsetl is mandatory, and surfsetr optional (which means surfsetl has to be used even for right hemisphere display only). The filename ends with .set, and is structured as follows:

The file has to be in the same directory as the surface and normal files (created with freesurfers mris_sconvert / mris_convert -n). Each line (except ones starting with#, which are ignored) has to contain the full surface filename (ending in .asc); the parameter (e.g. --surfaces/lh.set) gets truncated to get the path, and the filename is added.

In addition, three floats after the filename are interpreted as shift in the x,y,z direction. This shift can be necessary in order to separate e.g. the spherical representations of the cortical surfaces. For missing numbers, that component of the shift is 0.

Example for a .set-file:

#rh.smoothwm.asc

rh.pial.asc

rh.inflated.asc 100

rh.sphere.asc 200 100 100

Connection files:

Connections can be either full binary connectivity matrices (.bin) or indexed connections (.txt). For the binary files, the matrix has to be the full square of n*n float values for n nodes in the surface files. The values are read with single precision / little-endian byte order.

For the indexed connections, each line has to contain the indices of two connected nodes, and the connection strength between these nodes (that means each line looks like this: int int float). The first node on the surface has the number 0. For the display of connectivity between two hemispheres, the nodes on the left hemisphere start with 0, and the ones on the right hemisphere with (number of nodes on the left hemisphere). The connectivity is expected in one common file for both hemispheres, to enable inclusion of inter-hemispheric connections.

Options:

 --help: produce help message

  --surfsetl arg: (String) Filename for the first set of surfaces. Mandatory, used for display of one or two hemispheres, independently from which side of the brain is displayed

  --surfsetr arg: (String) Filename for a (optional) second set of surfaces

  --overlayl arg: (String) Filename of an optional overlay (.1D) file for the first hemisphere. Contains a single number (b&w) or three numbers (color) on each line. Number of lines has to be the number of nodes in the surface files.

  --overlayr arg: (String) Filename of an optional overlay (.1D) file for the second hemisphere.

  --scons arg: (String) Filename of connectivity (.bin) or indexed connection (.txt) files

  --config arg: (String) Filename of config file with options loaded at startup. Each option has to be on a separate line in the following shape: leftalpha = 0.5

  --leftalpha arg: (float) alpha (transparency) value between 0 and 1 for the first hemisphere

  --rightalpha arg: (float)alpha value for the second hemisphere

  --glyphs arg: (float)alpha value for the glyphs

  --threshold arg: (float)threshold for edges to be displayed

  --clipthreshold arg: (float) threshold determining how many connections are included from a full (.bin) connectivity matrix, does not have any effect on indexed connections. Choosing a low value with a big matrix can fill up the memory very quickly, and makes display slow

  --radius arg: (float) glyph radius

  --normalization arg: (float) normalization between 0 (no normalization) and 1 (completely normalized)

  --primsize arg: (float) size of display primitives (pointsize or linewidth, in pixels)

  --display arg: (int) number of surface in surface set that is displayed

  --color arg: (int) number of surface in surface set that determines the coloration of the glyphs

  --geometry arg: (int) number of surface in surface set that determines the geometric distribution of the glyph primitives

  --glyphstyle arg: 0 = geometry, 1 = projection, 2 = pie charts

  --primitives arg: 0 = points, 1 = vectors
